Git

คำสั่ง Git

git status ใช้เพื่อดูว่าตอนนี้มีไฟล์ไหนอัพโหลดแล้วและยังไม่อัพโหลด

git add <file> ใช้เพื่อขอเพิ่มไฟเป็นสถานะ stage (เตรียม commit)

git add . ใช้เพื่อขอเพิ่มไฟล์ทั้งหมดให้เป็นสถานะ stage (เตรียม commit)

git commit -m "massage ที่จะ Note การเปลี่ยนแปลงนี้" ใช้เพื่อสร้าง snapshot ให้ git เพื่อบันทึกว่ามีการแก้ไขอะไรและอัพเดตไฟล์ขึ้น git


git log --all --graph --decorate --oneline คือ การแสดง log ของการ commit git นั้น

git checkout <branch> คือ การเปลี่ยน branch

git checkout -b <new-branch> คือ การสร้าง branch ใหม่และเปลี่ยนไปที่ branch ใหม่นั้น

git merge <branch> คือการรวม branch ที่เจาะจงมายัง local

git remote add origin <url> คือการตั้งค่า remote เชื่อมไปที่ url ชื่อ origin

git remote origin คือเลือกว่าจะเชื่อมกับ url ที่ชื่อ origin

git push -u origin master ใช้สำหรับครั้งแรกในการ push ขึ้น github ครั้งต่อๆไปใช้แค่ git push ก็พอ

git push คือ ส่งการเปลี่ยนแปลงของไฟล์ไปบน remote repository - ใช้ในการอัพเเดตไฟล์ที่แก้ไข แต่ทุกครั้งที่แก้ไขไฟล์ต้องมีคำสั่งดังนี้ด้วย

git pull คือการ remote ไฟล์มายัง local (ดึงไฟล์ลงมาเพื่อให้ใน local มีไฟล์งานเป็นปัจจุบัน) โดยคำสั่ง git pull นั้นจะทำการ git fetch และ git merge ไปด้วย โดยเราจะมักเห็นใช้ git pull –rebase เพื่อทำการเปลี่ยนฐานแทนการ merge

git fetch คือ การตรวจสอบไฟล์ภายใน local และ remote ว่าตรงกันหรือไม่

git stash คือ การเก็บซ่อนการเปลี่ยนแปลงทั้งหมดไว้ทั้งหมด และสามารถนำกลับมาโดยใช้ git stash pop

git clone (url) คือการ คัดลอกโปรเจคจาก remote มายัง local
